# rapidcron 分布式任务调度系统 - 需求规格说明书（SRS）
**文档版本**：V1.0  
**编制日期**：2025年12月  
**适用场景**：本科毕业设计（底层分布式系统方向）  
**核心目标**：设计并实现一款轻量级、高可靠、秒级精度的分布式任务调度系统，支持跨语言执行、可视化管理与容器化部署，兼顾技术深度与工程实践完整性，满足中小型系统定时任务调度场景需求。

## 一、文档概述
### 1.1 文档目的
明确 rapidcron 系统的功能边界、技术指标、接口规范、部署要求及验收标准，为开发、测试、答辩验收提供唯一依据，同时体现分布式系统设计的核心思想与工程实现能力。

### 1.2 项目范围
| 包含范围 | 排除范围 |
|----------|----------|
| 1. 6字段Cron表达式解析（秒 分 时 日 月 周）及可视化编辑<br>2. 基于etcd的分布式锁与服务注册发现<br>3. 跨语言任务调度（Rust调度核心+Java执行端+Go执行端可选）<br>4. 任务全生命周期管理（创建/编辑/暂停/启动/删除/重试）<br>5. 执行日志记录、查询与导出<br>6. 核心监控指标（调度成功率、执行耗时、节点状态）可视化<br>7. 单机/集群部署（支持≤3个调度节点、≤5个执行节点）<br>8. Docker Compose一键部署与配置热更新<br>9. 基础权限控制（管理员/普通用户） | 1. 多区域灾备、异地多活架构<br>2. 大规模集群（＞10个节点）调度与扩容<br>3. 复杂权限体系（RBAC细粒度权限）<br>4. 海量日志存储（仅保留近90天日志）<br>5. 第三方云服务（如K8s、云监控）深度集成<br>6. 自定义调度算法（仅支持Cron标准调度）<br>7. 离线任务与流式任务处理 |

### 1.3 术语定义
| 术语 | 定义 |
|------|------|
| 调度节点（Scheduler Node） | 运行Rust调度核心的节点，负责任务解析、触发判断、分布式锁竞争、任务分发 |
| 执行节点（Executor Node） | 运行Java/Go执行端的节点，接收调度任务并执行，回传执行结果 |
| CronManager | 核心模块，负责Cron表达式解析、触发时间计算、时区转换（默认UTC+8） |
| 分布式锁 | 基于etcd实现，用于避免多调度节点重复触发同一任务 |
| 任务元数据 | 任务的核心配置信息（任务ID、名称、Cron表达式、执行节点、状态等） |
| 配置热更新 | 修改核心配置后无需重启服务，配置自动生效（生效延迟≤30秒） |
| UAT环境 | 验收测试环境，复用测试环境配置，用于模拟真实场景验证功能完整性 |

### 1.4 参考标准
1. Cron表达式6字段标准规范（秒 分 时 日 月 周）；
2. etcd v3.5+ 分布式锁与服务注册发现最佳实践；
3. gRPC v1.50+ 跨语言通信协议规范；
4. Docker Compose v2.0+ 容器编排标准；
5. RESTful API 设计规范（接口命名、错误码、响应格式）。

## 二、功能需求
### 2.1 核心调度模块
| 需求ID | 需求描述 | 验收标准 |
|--------|----------|----------|
| F001 | 支持6字段Cron表达式解析与验证 | 1. 正确解析标准表达式（如`0/5 * * * * *`每5秒、`0 0 9 ? * 1-5`工作日9点）<br>2. 支持特殊字符（`*`/`/`/`-`/`,`/`?`/`L`/`#`）<br>3. 非法表达式（字段数错误、数值越界、特殊字符非法）返回明确错误提示 |
| F002 | 触发时间计算（北京时间UTC+8） | 1. 单次触发时间计算误差≤1秒<br>2. 批量获取N次触发时间（1≤N≤100），返回有序时间列表（升序）<br>3. 跨天/跨月/跨年触发时间计算正确（如每月最后一天、每年1月1日） |
| F003 | 分布式锁机制 | 1. 多调度节点（≤3个）同时存在时，同一任务仅被一个节点分发<br>2. 锁超时时间可配置（默认30秒，范围10-120秒）<br>3. 锁竞争失败时无任务丢失，自动重试（重试次数≤3次） |
| F004 | 服务注册与发现 | 1. 调度节点/执行节点启动后自动注册至etcd，下线后自动注销（延迟≤10秒）<br>2. 调度节点可动态感知执行节点状态（在线/离线）<br>3. 执行节点离线时，任务自动分发至其他可用节点（节点数≥2时） |
| F005 | 任务触发与分发 | 1. 到达触发时间后，1秒内完成任务分发<br>2. 支持基于执行节点负载（CPU/内存使用率）分发任务（负载均衡策略可选）<br>3. 分发失败时触发重试（重试间隔：5秒、10秒、30秒），重试失败则标记任务为“分发失败” |

### 2.2 任务管理模块
| 需求ID | 需求描述 | 验收标准 |
|--------|----------|----------|
| F006 | 任务CRUD操作（支持UI/API两种方式） | 1. 创建任务：必填参数（名称、Cron表达式、执行节点、任务内容）校验通过方可提交<br>2. 编辑任务：仅允许修改未执行/暂停状态的任务，执行中任务禁止编辑<br>3. 删除任务：校验无正在执行的实例，有则提示“存在执行中实例，禁止删除”<br>4. 查询任务：支持按ID、名称、状态、执行节点筛选，支持分页（每页20/50/100条） |
| F007 | 任务状态管理 | 1. 支持状态：待执行（初始）、执行中、执行成功、执行失败、暂停、分发失败<br>2. 状态自动流转：待执行→执行中→执行成功/执行失败；暂停状态可手动启动→待执行<br>3. 失败任务支持手动重试（重试后状态重置为待执行） |
| F008 | 任务依赖配置（可选） | 1. 支持配置前置任务（1个任务最多依赖2个前置任务）<br>2. 仅当前置任务执行成功后，后置任务才可触发<br>3. 前置任务失败时，后置任务标记为“依赖失败”，支持手动重试 |
| F009 | 任务分片执行（可选） | 1. 支持按哈希/范围分片（分片数量1-10可配置）<br>2. 分片任务均匀分发至多个执行节点<br>3. 单个分片执行失败不影响其他分片 |
| F010 | 执行日志管理 | 1. 记录字段：任务ID、触发时间、执行节点、执行开始时间、执行结束时间、耗时、结果（成功/失败）、失败原因<br>2. 日志保留期限：默认90天，可通过配置修改（30-180天）<br>3. 支持按任务ID、时间范围、执行结果筛选日志，支持导出为CSV格式 |

## 五、验收标准
### 5.1 功能验收
1. 所有功能需求（F001-F019）100%通过测试用例验证，无阻塞性缺陷；
2. API接口（API001-API005）覆盖率100%，响应格式符合规范，错误码准确；
3. 核心场景验证通过：
   - 分布式锁验证（3个调度节点无重复调度）；
   - 节点故障恢复（关闭1个调度/执行节点，任务正常执行）；
   - 配置热更新（修改锁超时时间，30秒内生效）；
